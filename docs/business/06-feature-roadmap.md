## Feature Priority Framework

### How We Decide What to Build Next

**Score each feature 1-5 on:**

| Criteria | Weight | Question |
|----------|--------|----------|
| **Revenue Impact** | 30% | Will this drive new signups or upgrades? |
| **Retention Impact** | 25% | Will this reduce churn? |
| **User Demand** | 20% | Are vendors asking for this? |
| **Build Effort** | 15% | How long to build? (inverse — less effort = higher score) |
| **Strategic Value** | 10% | Does this create competitive moat? |

**Priority = weighted score.** Top 3 features per quarter get built.

### Feedback Channels
- In-app feature request button
- Monthly vendor survey (short — 3 questions max)
- Direct conversations with top vendors
- Support ticket pattern analysis
